# Comprehensive Testing

Generate thorough tests for the specified code.

## Coverage targets
1. **Happy path**: Normal expected usage
2. **Edge cases**: Boundaries, empty inputs, max values
3. **Error cases**: Invalid inputs, failures, exceptions
4. **Integration**: How it works with dependencies

## Test style
- Descriptive test names (should read like documentation)
- Arrange-Act-Assert pattern
- One assertion per test when possible
- Mock external dependencies
