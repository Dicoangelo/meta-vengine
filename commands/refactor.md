# Smart Refactor

Refactor code while maintaining behavior.

## Approach
1. **Understand**: Read and comprehend current implementation
2. **Test coverage**: Ensure tests exist (create if needed)
3. **Small steps**: Make incremental changes
4. **Verify**: Run tests after each change
5. **Clean up**: Remove dead code, improve naming

## Principles
- No behavior changes unless explicitly requested
- Prefer composition over inheritance
- Single responsibility per function/class
- DRY but don't over-abstract
