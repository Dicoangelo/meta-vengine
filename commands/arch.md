# Architecture Analysis

Analyze the codebase architecture and provide insights.

## Analysis areas
1. **Structure**: Directory layout, module organization
2. **Dependencies**: External packages, internal coupling
3. **Patterns**: Design patterns in use, consistency
4. **Data flow**: How data moves through the system
5. **Pain points**: Complexity hotspots, tech debt

## Output
- Visual diagram (ASCII or mermaid)
- Key components and responsibilities
- Recommendations for improvement
